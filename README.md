# Genshin Autoskip — авто-пропуск диалогов

Программа сама следит за экраном во время диалогов в Genshin Impact и
нажимает нужную клавишу вместо тебя:

- если внизу появилась иконка **"продолжить"** (маленький ромбик/стрелка под текстом реплики) — программа нажимает **Пробел**;
- если внизу появилась иконка с буквой **F** (выбор варианта ответа) — программа нажимает **F** (не важно, какой вариант выбрать).

Никакого взлома игры, чтения памяти процесса и т.п. — программа просто
смотрит на картинку экрана и жмёт клавиши, как это делал бы человек.

> ⚠️ Программа не делает ничего запрещённого (не читерит в бою, не фармит
> ресурсы), но любая автоматизация ввода формально может не нравиться
> разработчику игры. Используй на свой страх и риск и не запускай во время
> важных боевых/PvP-механик — предназначена только для диалогов.

---

## Шаг 1. Скачать программу

1. Открой страницу **Releases** этого репозитория на GitHub (справа на
   главной странице репозитория, или по ссылке вида
   `https://github.com/<владелец>/<репозиторий>/releases`).
2. У самого свежего релиза внизу будет файл **`GenshinAutoSkip.zip`** —
   скачай его.
3. Распакуй zip в любую папку (например, на рабочий стол) — правой кнопкой
   мыши по файлу → «Извлечь всё...».

Внутри будут файлы:

```
DialogueSkipper.exe      — сама программа автоскипа
CaptureScreenshot.exe    — помощник для настройки (делает скриншот экрана)
config.json              — настройки (область экрана, чувствительность)
README.md                — эта инструкция
templates/                — уже содержит 2 картинки-эталона (можно пересоздать, шаг 2)
```

### Windows ругается "Windows защитила ваш компьютер" (SmartScreen)

Это нормально — просто exe-файл не подписан цифровой подписью (это платно
и не нужно для личного использования). Нажми **«Подробнее»**, затем
**«Выполнить в любом случае»**.

---

## Шаг 2. Настройка перед первым запуском (один раз)

Иконки-эталоны (`templates/continue_icon.png` и `templates/f_icon.png`)
**уже лежат в архиве** — можно сразу переходить к пункту 2.3 (указать
область экрана) и пробовать запускать.

> ⚠️ Эти эталоны вырезаны из фотографии экрана телевизора, а не из чистого
> цифрового скриншота — на них может быть лёгкая рябь/муар от съёмки. Если
> после Шага 3 программа плохо ловит иконки (не срабатывает или, наоборот,
> путается), пересними свои собственные эталоны — это надёжнее:

### 2.1. (если нужно) Сделай свой скриншот

1. Зайди в игру, дождись любого диалога, чтобы на экране была видна иконка
   "продолжить" или "F".
2. Не выходя из игры (в оконном режиме или через Alt+Tab обратно),
   запусти **`CaptureScreenshot.exe`** двойным кликом.
3. Появится файл **`full_screenshot.png`** рядом с программой — это
   снимок твоего экрана целиком.

### 2.2. (если нужно) Вырежи свои иконки

1. Открой `full_screenshot.png` в программе **Paint** (есть в любой Windows:
   найди "Paint" через поиск в меню Пуск).
2. Найди на скриншоте иконку **"продолжить"** (маленький ромбик/стрелку под
   текстом последней реплики) — выдели её инструментом **"Выделить"**
   (Select) плотно, без лишнего фона, скопируй (Ctrl+C), вставь в новый файл
   Paint (Ctrl+N, затем Ctrl+V) и сохрани как:
   ```
   templates/continue_icon.png
   ```
   (перезаписав файл, который уже там лежит)
3. Так же вырежи иконку **квадратик с буквой "F"** (появляется при выборе
   варианта ответа) и сохрани как:
   ```
   templates/f_icon.png
   ```

### 2.2.1. Несколько вариантов одной иконки (если она не всегда ловится)

Иконки в игре могут накладываться на разный фон (яркий/тёмный, разные
цвета), из-за чего один-единственный эталон не всегда совпадает. Можно
добавить несколько вариантов одной и той же иконки, снятых в разных
ситуациях — программа сравнивает со всеми и берёт лучший результат.

1. Сохрани дополнительный вырезанный вариант рядом, например
   `templates/continue_icon_2.png`, `templates/continue_icon_3.png` и т.д.
2. Пропиши все файлы в `config.json` в списке `files`:
   ```json
   "templates": {
     "space": { "files": ["continue_icon.png", "continue_icon_2.png"], "key": "space" },
     "f": { "files": ["f_icon.png", "f_icon_2.png"], "key": "f" }
   }
   ```

Так же и с F — если она появляется на разной высоте экрана (например,
когда вариантов ответа несколько и они занимают несколько строк), просто
добавь ещё один вырезанный вариант в список.

### 2.3. Укажи область экрана

1. Снова открой `full_screenshot.png` в Paint.
2. Наведи курсор мыши на **левый верхний угол** зоны, где обычно появляются
   эти иконки (нижняя часть экрана) — в строке состояния внизу окна Paint
   Paint покажет координаты курсора в пикселях, например `900, 850`.
3. Наведи курсор на **правый нижний угол** этой же зоны — получишь вторую
   пару координат, например `1600, 1100`.
4. Открой файл **`config.json`** программой "Блокнот" (правой кнопкой →
   "Открыть с помощью" → "Блокнот") и впиши:
   ```json
   "region": {
     "top": 520,
     "left": 880,
     "width": 460,
     "height": 580
   }
   ```
   где:
   - `left`, `top` — координаты левого верхнего угла (из шага 3 выше);
   - `width` = (правая координата − left);
   - `height` = (нижняя координата − top).
5. Сохрани файл (Ctrl+S) и закрой Блокнот.

Область стоит брать **с небольшим запасом** вокруг иконок, чтобы не
промахнуться, но не захватывать сам текст диалога — иначе будет много
ложных срабатываний.

## Шаг 3. Запуск

1. Запусти игру в оконном режиме (или разверни на весь экран — главное,
   чтобы то, что видно на экране, совпадало с тем, что было на
   `full_screenshot.png`).
2. Запусти **`DialogueSkipper.exe`** двойным кликом.
3. Откроется чёрное окно с панелью — там видно, что программа запущена,
   сколько раз она нажала Пробел/F и последнее обнаруженное событие.
4. Сверни это окно (не закрывая!) и играй как обычно — во время диалогов
   программа сама будет жать нужные клавиши.

### Как остановить

Просто закрой окно программы (крестик) или нажми `Ctrl+C`, если оно в
фокусе.

## Если не работает

- **Ничего не происходит / кнопки не жмутся** — скорее всего, область
  экрана (`region` в `config.json`) указана неверно, или иконки в
  `templates/` вырезаны не с того разрешения. Повтори Шаг 2.
- **Слишком часто срабатывает не вовремя (ложные нажатия)** — увеличь
  `match_threshold` в `config.json`, например с `0.82` до `0.9`
  (чем ближе к `1`, тем строже совпадение).
- **Пропускает не всегда / иконка на одном фоне ловится, на другом нет** —
  добавь ещё один вариант эталона для этой же иконки (см. пункт 2.2.1) или
  немного уменьши `match_threshold` (например до `0.7`).
- **F появляется выше/ниже, чем захватывает `region`** (когда вариантов
  ответа несколько и подсказка "прилипает" к самому верхнему пункту) —
  увеличь `height` в `region`, чтобы зона захвата поднималась выше.
- **Программа находит иконку (score выше порога в панели), но клавиша не
  нажимается в игре** — попробуй запустить `DialogueSkipper.exe` от имени
  администратора, переключи в игре режим экрана на "Оконный"/"Оконный без
  рамки" вместо полноэкранного, и убедись, что окно игры активно (кликни по
  игре мышкой перед тем, как сворачивать программу).
- **Изменил разрешение экрана / масштаб интерфейса в игре** — нужно
  заново пройти Шаг 2 (иконки и координаты станут другого размера).

## Для разработчиков

Если хочешь запускать не exe, а сам код:

```bash
pip install -r requirements.txt
python capture_templates.py
python dialogue_skipper.py
```

Сборка exe вручную (то же самое, что делает GitHub Actions):

```bash
pyinstaller --onefile --console --name DialogueSkipper dialogue_skipper.py
pyinstaller --onefile --console --name CaptureScreenshot capture_templates.py
```

Новый релиз на GitHub (публикация тега/релиза) автоматически запускает
`.github/workflows/build-release.yml`, который собирает оба exe под Windows
и прикладывает `GenshinAutoSkip.zip` к релизу.
