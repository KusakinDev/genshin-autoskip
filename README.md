# Genshin Autoskip — авто-пропуск диалогов

Программа сама следит за экраном во время диалогов в Genshin Impact и
нажимает нужную клавишу вместо тебя. Логика в два шага:

1. Ищет на экране **золотую полосу-разделитель под именем персонажа** —
   она есть всегда, пока открыт диалог (и во время печати текста, и во
   время выбора варианта). Это большой, широкий элемент — случайно
   совпасть с чем-то посторонним (на рабочем столе, в игровом HUD) ему
   практически нереально.
2. Если эта полоса найдена — проверяет, есть ли рядом иконка с буквой
   **F** (выбор варианта ответа): если да — жмёт **F** (не важно, какой
   вариант выбрать), если нет — жмёт **Пробел** (это безопасно в любой
   момент монолога: если текст ещё печатается, игра просто домотает его
   мгновенно, если уже дописан — сразу продолжит).

Никакого взлома игры, чтения памяти процесса и т.п. — программа просто
смотрит на картинку экрана и жмёт клавиши, как это делал бы человек.

В любой момент можно поставить программу на паузу и снять с паузы, не
закрывая её — просто нажми **F9** (можно поменять в `config.json`, поле
`"toggle_hotkey"`). Хоткей работает глобально, даже если фокус на игре.

При каждом запуске программа сама проверяет, нет ли на GitHub версии
новее текущей — если есть, спросит `Скачать и установить сейчас? [y/N]`
прямо в консоли. Соглашаешься — скачает новый `DialogueSkipper.exe`/
`CaptureScreenshot.exe` и перезапустится сама. Твои `config.json` и
`templates/` (калибровка) при этом не трогаются.

> ⚠️ Программа не делает ничего запрещённого (не читерит в бою, не фармит
> ресурсы), но любая автоматизация ввода формально может не нравиться
> разработчику игры. Используй на свой страх и риск и не запускай во время
> важных боевых/PvP-механик — предназначена только для диалогов.

## Шаг 1. Скачать программу

1. Открой страницу **Releases** этого репозитория на GitHub (справа на
   главной странице репозитория, или по ссылке вида
   `https://github.com/<владелец>/<репозиторий>/releases`).
2. У самого свежего релиза внизу будет файл **`GenshinAutoSkip.zip`** —
   скачай его.
3. Распакуй zip в любую папку (например, на рабочий стол) — правой кнопкой
   мыши по файлу → «Извлечь всё...».

Внутри будут файлы:

```
DialogueSkipper.exe      — сама программа автоскипа
CaptureScreenshot.exe    — помощник для настройки (делает скриншот экрана)
config.json              — настройки (область экрана, чувствительность)
README.md                — эта инструкция
templates/                — уже содержит картинки-эталоны (можно пересоздать, шаг 2)
```

### Windows ругается "Windows защитила ваш компьютер" (SmartScreen)

Это нормально — просто exe-файл не подписан цифровой подписью (это платно
и не нужно для личного использования). Нажми **«Подробнее»**, затем
**«Выполнить в любом случае»**.

### Windows спрашивает разрешение (UAC, "Разрешить этому приложению вносить изменения?")

Это тоже нормально и ожидаемо — `DialogueSkipper.exe` **сам всегда
запрашивает права администратора** при запуске (без них нажатия клавиш
могут не доходить до игры). Нажми **«Да»**. Если случайно нажать «Нет» —
программа просто закроется, запусти её ещё раз.

## Шаг 2. Настройка перед первым запуском (один раз)

Эталоны (`templates/gate_bar.png`, `templates/gate_bar_2.png`,
`templates/f_icon.png`, `templates/f_icon_2.png`) **уже лежат в архиве** —
можно сразу переходить к пункту 2.3 (указать область экрана) и пробовать
запускать.

Если после Шага 3 обнаружение работает плохо (не срабатывает, или
наоборот — ловит то, чего нет), можно пересоздать эталоны под свой экран:

### 2.1. (если нужно) Сделай свой скриншот

1. Зайди в игру, дождись любого диалога, чтобы на экране было видно имя
   персонажа с золотой полосой под ним (и, если повезёт, выбор варианта
   с иконкой F).
2. Не выходя из игры (в оконном режиме или через Alt+Tab обратно),
   запусти **`CaptureScreenshot.exe`** двойным кликом.
3. Появится файл **`full_screenshot.png`** рядом с программой — это
   снимок твоего экрана целиком.

### 2.2. (если нужно) Вырежи свои эталоны

1. Открой `full_screenshot.png` в программе **Paint** (есть в любой Windows:
   найди "Paint" через поиск в меню Пуск).
2. Найди **тонкую золотую линию под именем персонажа** (сразу под жёлтым
   текстом с именем, например "Дэхья") — выдели её инструментом
   **"Выделить"** (Select) по всей ширине, но **не захватывая само имя**
   (нужна универсальная линия, которая одинаковая для любого персонажа),
   скопируй (Ctrl+C), вставь в новый файл Paint (Ctrl+N, Ctrl+V) и сохрани
   как:
   ```
   templates/gate_bar.png
   ```
   (перезаписав файл, который уже там лежит)
3. Так же вырежи иконку **квадратик с буквой "F"** (появляется при выборе
   варианта ответа) и сохрани как:
   ```
   templates/f_icon.png
   ```

### 2.2.1. Несколько вариантов одного эталона (если он не всегда ловится)

Эти элементы могут накладываться на разный фон (яркий/тёмный), из-за чего
один-единственный эталон не всегда совпадает. Можно добавить несколько
вариантов, снятых в разных ситуациях — программа сравнивает со всеми и
берёт лучший результат.

1. Сохрани дополнительный вырезанный вариант рядом, например
   `templates/gate_bar_3.png` или `templates/f_icon_3.png`.
2. Пропиши файл в соответствующем списке в `config.json`:
   ```json
   "gate_files": ["gate_bar.png", "gate_bar_2.png", "gate_bar_3.png"],
   "f_files": ["f_icon.png", "f_icon_2.png", "f_icon_3.png"]
   ```

### 2.3. Укажи область экрана

1. Снова открой `full_screenshot.png` в Paint.
2. Наведи курсор мыши на **левый верхний угол** зоны, где обычно появляются
   имя персонажа с полосой и иконка F (нижняя половина экрана) — в строке
   состояния внизу окна Paint появятся координаты курсора в пикселях,
   например `880, 520`.
3. Наведи курсор на **правый нижний угол** этой же зоны — получишь вторую
   пару координат, например `1340, 1100`.
4. Открой файл **`config.json`** программой "Блокнот" (правой кнопкой →
   "Открыть с помощью" → "Блокнот") и впиши:
   ```json
   "region": {
     "top": 520,
     "left": 880,
     "width": 460,
     "height": 580
   }
   ```
   где:
   - `left`, `top` — координаты левого верхнего угла (из шага 2 выше);
   - `width` = (правая координата − left);
   - `height` = (нижняя координата − top).
5. Сохрани файл (Ctrl+S) и закрой Блокнот.

Область стоит брать **с запасом** — она должна охватывать и место, где
появляется имя персонажа с полосой, и место, где появляется F (они на
разной высоте, особенно если вариантов ответа несколько).

## Шаг 3. Запуск

1. Запусти игру в оконном режиме (или разверни на весь экран — главное,
   чтобы то, что видно на экране, совпадало с тем, что было на
   `full_screenshot.png`).
2. Запусти **`DialogueSkipper.exe`** двойным кликом.
3. Откроется чёрное окно с панелью — там видно текущий score полосы-gate
   и иконки F, сколько раз программа нажала Пробел/F, и последнее
   обнаруженное событие.
4. Сверни это окно (не закрывая!) и играй как обычно — во время диалогов
   программа сама будет жать нужные клавиши.

### Пауза/продолжить без закрытия

Нажми **F9** в любой момент (даже когда окно программы свёрнуто, а фокус
на игре) — статус в панели переключится между "работает" и "на паузе".
Удобно, если хочешь на время боя/фарма гарантированно не получать
случайных нажатий, не завершая процесс.

### Как остановить

Просто закрой окно программы (крестик) или нажми `Ctrl+C`, если оно в
фокусе.

## Если не работает

- **Ничего не происходит / кнопки не жмутся** — скорее всего, область
  экрана (`region` в `config.json`) указана неверно, или эталоны в
  `templates/` вырезаны не с того разрешения. Повтори Шаг 2, следи за
  панелью — она в реальном времени показывает score полосы (`Gate`) и
  иконки F, сразу видно, промахивается регион или нет.
- **Слишком часто срабатывает не вовремя (ложные нажатия)** — увеличь
  `gate_threshold` в `config.json`, например с `0.7` до `0.85`.
- **Пропускает не всегда / эталон на одном фоне ловится, на другом нет** —
  добавь ещё один вариант эталона (см. пункт 2.2.1) или немного уменьши
  соответствующий порог (`gate_threshold` / `f_threshold`).
- **F появляется выше/ниже, чем захватывает `region`** (когда вариантов
  ответа несколько и подсказка "прилипает" к самому верхнему пункту) —
  увеличь `height` в `region`, чтобы зона захвата поднималась выше.
- **Программа находит нужное (score выше порога в панели), но клавиша не
  нажимается в игре** — попробуй запустить `DialogueSkipper.exe` от имени
  администратора, переключи в игре режим экрана на "Оконный"/"Оконный без
  рамки" вместо полноэкранного, и убедись, что окно игры активно (кликни по
  игре мышкой перед тем, как сворачивать программу).
- **Изменил разрешение экрана / масштаб интерфейса в игре** — нужно
  заново пройти Шаг 2 (эталоны и координаты станут другого размера).
- **F9 не реагирует** — запусти `DialogueSkipper.exe` от имени
  администратора: глобальный хоткей иногда не перехватывается без
  повышенных прав, особенно если игра сама запущена от администратора.

## Для разработчиков

Если хочешь запускать не exe, а сам код:

```bash
pip install -r requirements.txt
python capture_templates.py
python dialogue_skipper.py
```

Сборка exe вручную (то же самое, что делает GitHub Actions):

```bash
pyinstaller --onefile --console --name DialogueSkipper dialogue_skipper.py
pyinstaller --onefile --console --name CaptureScreenshot capture_templates.py
```

Новый релиз на GitHub (публикация тега/релиза) автоматически запускает
`.github/workflows/build-release.yml`, который собирает оба exe под Windows
и прикладывает `GenshinAutoSkip.zip` к релизу.

Константу `VERSION` в `dialogue_skipper.py` вручную поднимать не нужно —
шаг `Set VERSION from release tag` в workflow сам подставляет в неё тег
публикуемого релиза перед сборкой (`v1.2.0` → `VERSION = "1.2.0"`), именно
с ней потом сравнивает себя автообновление у пользователей. Значение
`VERSION`, закоммиченное в репозитории, — просто дефолт для локальной
сборки/разработки, на реальных релизах оно всегда переписывается.
